# 项目简介

## 项目背景

日前小铁、无忧存分为2个品牌2套系统，用户独立单独运营；为后续平台化建设及持续商业化，现将线上线下两个平台融合为一个平台；

## 项目目标

### 总体目标

搭建小铁寄存服务平台，线上线下以及行李配送等业务集成在一个小程序，进入平台化运营阶段；

### 阶段目标

| 阶段计划 | 阶段目标 | 截止时间 |
| --- | --- | --- |
| 寄存业务合并 | 完成小程序线上预约、线下扫码寄存业务合并，功能发布到无忧存小程序；完成寄存功能的验证，寄存支持平台卡券； | 2025年12月31日 |
| 行李配送业务合并 | 配送业务合并，通过平台可下单、预约轻松游配送服务； | 2026年1月31日 |
| 迭代阶段 | 广告投放等业务合并 | —— |

## 功能兼容

| **功能点** | **需求平台** | **兼容方案** |
| --- | --- | --- |
| 强制关注公众号 | 小铁寄存柜 | 场景限制，仅在扫码下单流程出现 |
| 无忧存线下渠道码贴纸 | 无忧存 | 链接不变，功能开发兼容 |
| 获取定位权限 | 无忧存 | 进入首页才提醒授权，寄存柜流程不提醒 |
| 公众号、小程序对接对接 | 小铁寄存柜 | 这部分不切，还是用小铁寄存柜 |
| 跨平台合作接口对接 | 无忧存 | 暂时搁置，重新整理广告位后对接 |
| 新用户送折扣券 | 无忧存 | 重新设计平台用户运营方案 |
| 历史无忧存账户卡券 | 无忧存 | 仅支持门店寄存使用 |
| 邀请好友送卡券 | 无忧存 | 二期需求 |
| 寄存柜消息推送 | 小铁寄存柜 | 增加小铁寄存柜小程序推送，不同流程授权不同消息通知； |
