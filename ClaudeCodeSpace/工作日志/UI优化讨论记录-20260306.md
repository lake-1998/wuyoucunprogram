## 一、今日讨论结论

### ✅ 达成共识

| 结论 | 说明 |
|------|------|
| 智能场景识别方向正确 | 扫码场景 vs 搜索场景差异化首页整体认可 |
| 场景A（扫码进入）方案可行 | 摄像头前置，搜索降级 |
| 场景B（搜索进入）方案可行 | 搜索框置顶，附近网点上移 |

### ❌ 需要修改的内容

**1. 场景C（首次访问引导弹窗）建议删除**

- **问题**：弹窗问用户"是要扫码还是找网点"，在首次访问时用户根本不了解区别，是把决策成本转嫁给用户
- **数据支撑**：无忧存搜索用户60%+，首次用户几乎都来自搜索，无需二选一弹窗
- **用研结论**（Marcus）：首次访问弹出选择框会触发"焦虑-退出"反应，尤其旅途中时间紧迫的用户
- **建议改法**：删除场景C弹窗，默认展示搜索优先布局（场景B），扫码作为常驻次要入口保留在首页

**2. 用户偏好记忆功能（localStorage）需降优先级**

- **技术风险**（Wei Zhang）：微信小程序 Storage 在卸载重装后清空，目标用户3-6个月才用一次，大概率每次都像新用户
- **建议**：改为服务端存储（需绑定手机号登录），或直接降为P2甚至移除该功能

**议题1：登录态对UI展示的系统性影响（未完成讨论）**

- **提出者**：Annie Chen
- **背景**：方案中多处涉及"登录后显示精确位置"，但全文没有系统梳理登录态影响的UI变化
- **需要补充**：梳理未登录/已登录/已下单三种状态下，各关键页面的UI差异对照表
- **关键问题**：Dana Liu 提出——**当前无忧存的登录用户占比是多少？**（lake 待回答）
  - 如果登录率 < 30%，则"登录后显示精确位置"方案覆盖率极低，需重新评估优先级

**议题2：场景识别的前端技术实现方案（未展开）**

- **提出者**：Wei Zhang
- **待讨论**：智能场景识别（扫码 vs 搜索 vs 直接打开）在前端层面的具体实现

  - 渠道码逻辑如何兼容？
  - 微信/支付宝/抖音三端实现是否一致？

**议题3：方案中登录引导时机设计（未展开）**
