# 无忧存用户端版本合并 UI 优化方案

**文档版本**: v1.0
**创建日期**: 2026年3月6日
**文档用途**: 小铁寄存柜 + 无忧存平台合并 · 用户端C端UI全页面优化方案
**设计基准**: 小铁黄主色调 + 小铁UI组件规范

---

## 一、设计原则与总体方向

### 1.1 合并后的设计核心矛盾

| 维度 | 小铁寄存柜（线下） | 无忧存（线上） | 合并后策略 |
|------|-----------------|--------------|---------|
| 访问来源 | 扫码80%+ | 搜索60% | **智能场景识别，动态切换布局** |
| 核心诉求 | 直接开柜存包 | 搜索地标找网点 | **首屏分场景优先级** |
| 用户画像 | 旅游+商务人群 | 30岁以内60% | **年轻化但不失专业感** |
| 信任需求 | 柜子在眼前，不需要 | 线上决策，需要信任背书 | **详情页增强可信度** |

### 1.2 四大设计原则

1. **场景优先**：通过来源判断调整首屏功能权重，减少无关认知负担
2. **关键路径极简**：从进入到完成寄存的主路径点击不超过4步
3. **信任前置**：关键转化节点提前展示安全保障（保险、评价、位置）
4. **双端统一**：微信/支付宝/抖音小程序视觉统一，体验一致

### 1.3 设计规范

- **主色调**：小铁黄 `#FFB300`（按钮、强调色、底部导航选中态）
- **辅色**：深灰 `#333333`（主文字）、中灰 `#999999`（辅助文字）、浅灰 `#F5F5F5`（背景）
- **组件库**：沿用小铁寄存柜 UI 组件，融合无忧存业务功能
- **字体**：主文字16px，副文字13px，注释12px
- **圆角**：卡片8px，按钮24px（pill形）

## 二、首页优化方案（核心）🔴

> 首页是最大问题区域，影响从进入到下单的全链路转化。

### 2.1 当前问题诊断

```

┌─────────────────────────────────┐
│  顶部Banner（可配置）            │
├─────────────────────────────────┤
│  ┌─────┐ ┌─────┐ ┌─────┐        │  ← ❌ 3模块平铺，视觉权重相同
```

**核心问题**：

- "我的附近"在页面下方，点击率只有4.66%，与搜索差距13倍
- 两种用户场景（扫码用户/搜索用户）混在同一首页无差异化

### 2.3 首页搜索区优化详情

**优化前**：4个入口（定位 / 热门搜索点 / 更多 / 搜索框）都跳同一地标输入页

**优化后**：统一为2条路径

| 路径 | 入口 | 目标 | 说明 |
|------|------|------|------|
| 快速找点 | 搜索框 → 输入地标 → 地图页 | 精准搜索 | 合并原来4个入口 |
| AI推荐 | [AI]按钮 → 对话框 | 自然语言推荐 | 独立保留，不与搜索混淆 |

**热门地标标签规则**：
- 显示3个固定标签（按配置优先级）
- 去除"更多>"按钮（原来是多余跳转）
- 用户搜索后，搜索框下方展示历史记录（最近3条）

### 4.2 防爬虫策略优化（核心改动）

**问题**：当前策略是下单才能看位置/电话 → 用户不敢下单 → 恶性循环

**推荐方案（分三档展示）**：

| 用户状态 | 位置显示 | 电话显示 | 防爬效果 |
|---------|---------|---------|---------|
| 未登录 | 模糊地址+地图圆圈（500米范围） | 不显示 | ⭐⭐⭐ |
| 已登录未下单 | 精确地址+精确地图标注 | 虚拟号 | ⭐⭐⭐⭐ |
| 已下单 | 精确地址+精确地图标注 | 真实号码 | ⭐⭐⭐⭐⭐ |

**技术防爬补充**：
- 已登录用户每小时查看网点上限50个（普通旅行者最多查3-5个）
- 超出频率触发验证码
- 设备指纹识别爬虫行为

```
订单聚合页布局：
```
